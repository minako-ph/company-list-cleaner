## 本プロダクトが使う機能（CR-1/CR-2により「登録番号を指定して情報を取得する機能」のみ）

```
GET https://web-api.invoice-kohyo.nta.go.jp/1/num
    ?id=<アプリケーションID(13桁)>
    &number=<T+13桁>[,<T+13桁>...]   # カンマ区切り・最大10件
    &type=21                          # 01=CSV / 11=XML / 21=JSON（文字コードはUnicode）
    &history=0                        # 0=履歴なし(既定) / 1=登録・取消・失効の履歴あり
```

- **1リクエスト最大10件**（超過は 400 エラーコード0002）→ サイドバーの50行バッチは 10件×5リクエスト に分割（N-1の直列キュー内で処理）。
- バージョン値は「1」固定（パス先頭）。
- 「取得期間を指定して情報を取得する機能（/diff）」「登録番号と日付を指定して情報を取得する機能（/point）」は存在するが、**申請書記載の照会形態と一致する /num 以外は使用しない**。

## 提供時間帯

- 固定の提供時間の定めなし。「本機能の利用可能時間は、本機能が停止する以外の時間」（利用規約第8条）。
- メンテナンス・非常事態・その他必要時に停止あり（事前告知は公表サイト掲載。緊急時は無告知）→ N-4監視＋サイドバー障害表示、N-7縮退で対応。

## レート・アクセス制限

- 数値上限の明記なし。禁止事項に「短時間における大量アクセスその他本機能の運用に支障を与えること」（利用規約第9条一項三号）。利用集中時は国税庁側が利用制限可（第8条3項）。
- → 既定 `RATE_RPS=1`（プロセス内直列キュー）を維持。調整の根拠が出るまで上げない。

## データ更新頻度

- 公表情報の更新は1日1回、登録簿登載の翌開庁日 午前6時目安（休日除く）。
- ※CR-3（都度取得・都度破棄）によりキャッシュ最適化には使えない。参考情報のみ。

## アプリケーションID・検証環境

- アプリケーションIDは法人番号システムWeb-APIと**統合（共通ID）**。本番・検証環境の両方で同一IDを使用可。
- 検証環境: 令和3年10月より提供。架空の個人・法人サンプルデータで応答。**接続先URLは公開仕様書に非掲載**→ID発行時の案内で確認する（TODO。判明したら `INVOICE_API_BASE` に設定し decisions.md に追記）。

## 主なエラーコード（/num 関連）

| HTTP | コード | 内容 |
|---|---|---|
| 400 | 0001 | 登録番号未指定 |
| 400 | 0002 | 登録番号が10件超 |
| 400 | 0003 | 登録番号n の値が空 |
| 400 | 0004 | 登録番号n が14桁でない |
| 400 | 0005 | 登録番号n が「T＋数字13桁」でない |

## 利用規約上の実装上の注意（CR-6関連）

- 出典明示義務: 「このサービスは、国税庁適格請求書発行事業者公表システムのWeb-API機能を利用して取得した情報をもとに作成しているが、サービスの内容は国税庁によって保証されたものではない」を適宜の場所に明示（第7条関連）→ 引継書§8のverbatim定数と一致。
- IDの第三者への譲渡・貸与・開示の禁止（第9条一項五号）→ アプリIDはバックエンドのSecret Managerのみに保持（§9）。
